Overlay Zones

Trendy marketing name TBD...

Zones

a collection of data and links in a hierarchical tree

authoritatively owned and editable by an associated Eth address

uniquely identified by a content addressed hash (of the contents of the zone)

Publishing

smart contract associates owned eth address with a zone hash

queryable by address to find current zone for a given address

updates to zone content will update the associated zone hash

What kind of content?

URLs, images, documents, public keys - basically a generic file system

Data is stored off chain using conventional means - file systems, DBs, web servers, S3 whatever

in addition you can store links to other zones - what we call an overlay

Overlays

allow you to reference another zone in whole or part at any point in your own zone

other zones' data transparently merged into your own, including duplicates

crucially ownership of overlayed zones remains with the original zone's owner

Why?

reference other author's content whom you trust or want to follow

compose data from other schemas into your own zone with whatever structure you need

maintenance and upkeep remain with other authors - their changes immediately visible once published

Applications

the usual directory suspects: domain names, addresses, public keys, CRLs, etc

collaborative content aggregation, curated lists, indices

materializing disparate sources of truth into a coherent entity
